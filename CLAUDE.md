# CLAUDE.md

## Exports / reports

Any data export or report generated by this app (ScoutBoard, portfolio reports, search results, etc.) must be a styled Excel (.xlsx) workbook — never a plain CSV or raw data dump. Use `src/lib/excelStyle.ts` (shared teal/gold-branded header, zebra rows, wrap text, page setup) as the base:

- Single-table exports: `src/lib/simpleWorkbook.ts` → `buildSimpleWorkbook(...)`
- Multi-sheet reports: see `src/lib/portfolioWorkbook.ts` for the pattern (one sheet per section/facility, shared/overlap sheets highlighted).

Dynamically `import()` these modules at the export call site (not a static top-level import) — `exceljs` is large and must stay out of the main bundle, loading only when an export is actually triggered.

Every export should read as a clean, pretty, presentable report on its own — not just data thrown into rows.

## No disclaimer/explanation captions in the UI

Default to zero muted caption paragraphs under section headers explaining what a section means, how to use a control, or caveating the data (e.g. "Gold-ringed pins are your portfolio's facilities..."). The UI must be self-explanatory through clear labels, structure, and design first — not through prose disclaimers. If a control's meaning isn't obvious from its label/layout, fix the label or layout; don't bolt on an explanation. This applies everywhere, not just one screen.

This is a strong default, not an absolute ban: if a piece of context is genuinely non-obvious and actually necessary (can't be inferred from labels/structure) and isn't already stated elsewhere on screen, it's fine to keep — just make it as unobtrusive as possible (e.g. a `title` hover tooltip on the relevant label rather than a persistent visible sentence) instead of reflexively deleting real information. Don't keep anything that's redundant with something already visible (e.g. a per-row "as of" tag makes a blanket footer saying the same thing redundant — cut the footer). Empty-state messages (e.g. "No saved deals yet") are fine — those aren't disclaimers, they explain an otherwise-blank screen.
